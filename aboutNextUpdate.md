
# Next Update (1.1.0) / 다음 업데이트 (1.1.0)

---

1. I am updating the database (MySQL, SQLite) for application
   -> **Database Engine Update (MySQL, SQLite Implementation):** This will significantly **improve data processing speed and stability** in large-scale server environments.
   
   -> **데이터베이스 엔진 업데이트 (MySQL, SQLite 적용):** 이를 통해 **대규모 서버 환경에서 데이터 처리 속도와 안정성이 크게 향상**됩니다.

2. The debug feature is being extended.
   -> **Debug Feature Extension:** New log flags, such as `logEventTrigger`, will be added, allowing for **more precise tracking of the root cause of issues**.
   
   -> **디버그 기능 확장:** `logEventTrigger`와 같은 새로운 로그 플래그가 추가되어 **문제 발생 시 원인을 더욱 정밀하게 추적**할 수 있습니다.

3. A simple exception is being processed.
   -> **Simple Exception Handling Enhancement:** Stability will be reinforced by ensuring the **server does not crash but only provides a warning to administrators** in unforeseen circumstances (e.g., malformed JSON files).
   
   -> **단순 예외 처리 보강:** 잘못된 JSON 포맷 등 예기치 않은 상황에서 **서버가 다운되지 않고 관리자에게 경고만 제공**하도록 안정성을 강화합니다.

# Next Update (1.2.0) / 다음 업데이트 (1.2.0)

---

1. Various new events are added.
   -> **Multiple Event Expansion:**
   Support for **a wide variety of in-game events** (such as player actions, block-related events, and entity interactions) will be added, allowing **more flexible and extensible system configurations**.

   -> **다양한 이벤트 추가:**
   플레이어 행동, 블록 관련 이벤트, 엔티티 상호작용 등 **다양한 인게임 이벤트가 추가**되어, **시스템 확장성과 설정 자유도가 크게 향상**됩니다.

2. NBT functionality is now supported.
   -> **NBT Feature Support:**
   Item conditions and rewards can now be defined using **NBT data**, enabling **precise item identification and advanced customization** beyond simple material-based checks.

   -> **NBT 기능 사용 가능:**
   **NBT 데이터를 기반으로 아이템 조건 및 보상 설정이 가능**해져, 단순한 아이템 타입 비교를 넘어 **정밀한 아이템 판별과 고급 커스터마이징**이 가능합니다.
