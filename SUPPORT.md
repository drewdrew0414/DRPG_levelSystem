## 1. Bug Reports
If you discover a bug or encounter an unexpected error, please report it by following these steps. Detailed reporting significantly helps us in troubleshooting.

### A. Self-Diagnosis (Diagnostic Check)
 * Enable Debug Mode: Set Debug.useDebug to true in config.json and reload your server to check the console logs.
 * Integrity Check: Use the administrator command /drpg validate to ensure there are no errors in your JSON configuration files.
 * Review Log File: Check the separate log file (rpg-log.txt), created based on the Logging options in the README.md, to locate any specific error messages. *
   
### B. Official Reporting Channels
 * Recommended Channel: GitHub Issues
   * This is the preferred method for reporting. Please include detailed steps to reproduce the bug, the full stack trace, and your server environment (e.g., Paper version).
 * Email: drew0414drew@gmail.com
   * If you are unable to use GitHub, please send a detailed description of the error along with any relevant log files attached.

## 1. 버그 보고 (Bug Reports)

버그를 발견하셨거나 예상치 못한 오류가 발생했을 경우, 가능한 한 다음 절차에 따라 보고해 주시면 문제 해결에 큰 도움이 됩니다.

### A. 자가 진단 (Diagnostic Check)

1.  **디버그 모드 활성화:** `config.json`에서 `Debug.useDebug`를 `true`로 설정하고 서버를 재로드하여 로그를 확인해 주십시오.
2.  **무결성 검사:** 관리자 명령어 `/drpg validate`를 사용하여 JSON 설정 파일에 오류가 없는지 확인해 주십시오.
3.  **로그 파일 확인:** `README.md`의 로깅 옵션에 따라 생성된 별도 로그 파일(`rpg-log.txt`)을 확인하여 오류 메시지를 찾아 주십시오.

### B. 공식 보고 채널
